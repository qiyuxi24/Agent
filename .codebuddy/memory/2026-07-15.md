# 2026-07-15

## 前端配色方案重构 — 三层 Token 体系

### 背景
现有配色方案只有 20 个 CSS 变量，硬编码颜色散布在 8 个 CSS 文件中（~30+ 处 `#xxxxxx`，大量 `rgba()` 直接引用颜色值）。

### 实施内容
1. **重写 `variables.css`**（60→~260行）：三层体系
   - Layer 1 Primitives：原始色板（indigo/red/green/amber/purple/sky/blue/dark/gray + code block），跨主题不变
   - Layer 2 Semantic：语义映射（accent/success/danger/error/warning/info/brand/ide/stderr/env-chip），按主题切换
   - Layer 3 Component：组件 token（bg/text/border/shadow/radius/status overlays），消费端唯一引用
   - 向后兼容：保留 `--accent`/`--danger`/`--accent-hover` 等旧名作为语义层别名

2. **清除所有硬编码颜色**（7 个文件，~50 处替换）：
   - `chat.css`：代码块 `#282c34`→`--code-header-bg`，时间线 `#4ade80`→`--color-success`，`#ffffff`→`--text-on-accent`
   - `mcp.css`：状态色 `#4ad88a`→`--color-success`，错误 `#ff6b6b`→`--color-error`，stderr→`--color-stderr`，清理全部 var() fallback 硬编码
   - `rag.css`：成功绿 `#22c55e`→`--color-success`，danger overlay→`--danger-bg/border`
   - `settings-forms.css`：主题预览→`--palette-white/dark-700`，badge 色→`--color-success-dark/brand/warning`
   - `code-server-status.css`：IDE 蓝渐变→`--color-ide/ide-light/ide-progress`，错误渐变→`--color-danger/danger-hover`
   - `buttons.css`/`layout.css`：`#ffffff`→`--text-on-accent`
   - `settings-provider.css`/`workspace.css`：danger hover→`--danger-bg`

3. **创建 `src/tokens/theme.ts`**：TS 侧颜色常量，镜像 CSS token。导出 `palette` / `semanticLight` / `tokensLight` / `tokensDark`。

### 效果
- 所有 CSS 文件（除 variables.css）零硬编码十六进制颜色
- 改主色调只需改 variables.css Layer 1 中 indigo 色阶
- 新增主题只需复制 `.theme-dark` 块并修改 Layer 2-3 映射
- TS 侧可通过 `import { tokensLight } from '@/tokens/theme'` 获取颜色值
