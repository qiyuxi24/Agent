## Agent Loop 增强：补齐与主流方案的差距（2026-07-16）

**差距分析**（对照 OpenAI Agents SDK / LangGraph / Anthropic Claude）：
- 缺少上下文窗口管理 → 无限累积，无 token 估算/压缩
- 缺少实时流式 → 先缓冲再回放，用户看不到实时思考
- 缺少工具使用行为配置 → 固定 run_llm_again
- 缺少人工审批 → 无法在敏感工具前暂停等待确认
- 缺少 Token 追踪 → 无法排查上下文超限

**修改文件（3 个）**：

1. `src-tauri/src/types.rs` — 新增 4 个事件类型：
   - `ContextCompacted`（上下文压缩通知）
   - `ToolApprovalRequired` / `ToolApprovalDecision`（人工审批流）
   - `TokenUsageEvent`（每轮 Token 统计）

2. `src-tauri/src/agent_loop.rs` — 全面重写，新增功能：
   - **上下文窗口管理**：Token 估算（chars/3）→ 达到 80% 阈值时自动滑动窗口摘要压缩，保留 system prompt + 最近 4 条 + 最新用户消息
   - **实时流式思考**：`parse_sse_response` 中 `stream_content=false` 时 content 实时推为 `thinking-delta`（不等待 LLM 返回）
   - **ToolUseBehavior** 枚举：`RunLlmAgain`（默认）/ `StopOnFirstTool` / `StopAtTools(Vec<String>)`
   - **HITL 审批流**：`LoopContext.approval_rx` 通道，`require_tool_approval_for` 列表指定需审批工具，前端通过 `tool_approval_response` 命令回传决策
   - **Token 追踪**：每轮 LLM 调用后 emit `token-usage` 事件
   - **工具结果富化**：`enrich_tool_result()` 对 >5K 字符的纯文本结果自动摘要，JSON 结果结构化压缩（数组截 50 项/字符串截 200 字符）
   - 保留所有旧 API（LlmClient trait / ToolExecutor / LoopContext / run_agent_loop 签名不变）

3. `src-tauri/src/lib.rs` — 改动：
   - `parse_sse_response`：`stream_content=false` 时 content 实时推为 `thinking-delta`；非推理模型首次 content 到达时自动 emit `thinking-start`
   - 新增 `tool_approval_response` Tauri 命令
   - `AppState` 新增 `active_approval_streams`
   - `chat_stream` 创建审批通道并传入 LoopContext
   - config 增加 `context_limit=128K` / `enrichment_threshold_chars=5000`

**测试**：14 个 agent_loop 测试全部通过（6 个旧测试 + 8 个新测试覆盖：StopOnFirstTool / StopAtTools / context_compaction / 4 个 token 估算/压缩/富化/JSON 压缩）

## Agent Loop 增强 V2：L2 验证 + 请求透传 + 超时保护（2026-07-16）

**背景**：参考 2026 年 Loop Engineering 最新实践（Anthropic Loop Engineering、Addy Osmani 六大组件、LangChain 四层循环堆叠），在已有 L1 Agent Loop 基础上补齐。

3. `src-tauri/src/lib.rs` — chat_stream 透传 ChatRequest 字段到 AgentLoopConfig

**新增功能**：
- **请求参数透传**：`ChatRequest` 新增 `tool_use_behavior`/`stop_at_tool_names`/`require_tool_approval_for`/`max_iterations`/`enrichment_threshold_chars`/`llm_timeout_secs`，前端 Agent 配置可直接控制 Agent Loop 行为
- **L2 验证循环**：`verify_output()` — 主循环结束后调用 LLM-as-Judge 检查回答质量（Maker/Checker 分离原则），验证失败则追加一轮自修正。由 `AgentLoopConfig.enable_verification` 控制，默认关闭
- **单次 LLM 超时保护**：`llm_timeout_secs` 通过 `tokio::time::timeout` 包装每轮 LLM 调用，防止单次调用挂太久
- **结构化轮次跟踪**：`agent-iteration` 事件（`AgentIterationEvent`：iteration/total/phase/elapsed_ms），`agent-loop-stats` 事件（`AgentLoopStats`：总轮次/耗时/工具调用数/压缩次数/是否验证），`agent-iteration-tick`（向后兼容旧格式）

**测试**：14/14 agent_loop 单测通过，零 lint 错误
