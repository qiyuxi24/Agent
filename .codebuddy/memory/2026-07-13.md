# 2026-07-13

## 排查并修复编译/可移植性问题

### 问题诊断
排查了项目在其他开发者机器上无法编译的根因，发现下列问题并修复：

**P0 阻塞级：**
1. **Node.js v24 ↔ code-server 要求 Node 22**：code-server v4.127.0 的 engines 指定 node 22，本机用 v24 导致原生模块不兼容
2. **PROTOC 路径硬编码**：`.cargo/config.toml` 中 PROTOC 指向用户特定路径 `C:\Users\34239\...`，其他机器必然失败
3. **--ignore-scripts 跳过原生编译**：`download-code-server.mjs` 和 `build.rs` 都用 `--ignore-scripts` 安装 code-server，导致 7 个 `@vscode/*` .node 文件缺失时无法自动修复
4. **tauri.conf.json $schema 错误**：指向 nicegui 而非 tauri 官方 schema

**P1 限制级：**
5. 无 `.nvmrc` / `engines` 版本约束
6. `build.rs` 与 `download-code-server.mjs` 重复下载逻辑
7. README 缺少构建前提说明（VS BuildTools、Spectre 库、protoc 等）
8. `start-dev.bat` 不下载 code-server，IDE 功能静默失败
9. `release.bat` typo: "taudi.conf.json"

### 修复内容
- **新增 `.nvmrc`** → 锁定 Node 22
- **package.json 新增 `engines`** → `"node": ">=18 <=22"`
- **`.cargo/config.toml`** → PROTOC 加注释指引，保留本地路径供当前用户使用
- **`tauri.conf.json`** → $schema 改为 tauri 官方 URL
- **`download-code-server.mjs`** → 新增原生模块检测+自动编译步骤（7个 .node 文件逐一检查）
- **`build.rs`** → 新增 `check_native_modules()` 函数，缺失时打印修复指引
- **`start-dev.bat`** → 新增 code-server 检查和自动下载
- **`README.md`** → 补充完整构建前提 + 故障排查章节
- **`release.bat`** → 修复 typo
